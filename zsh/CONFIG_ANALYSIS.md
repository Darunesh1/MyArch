# HyDE Zsh Configuration Analysis

Complete analysis of the HyDE (Hyprland Dotfiles) zsh configuration structure.

---

## 📁 Directory Structure

```
~/.config/zsh/
├── .zshenv              # Entry point - loads conf.d/*.zsh
├── .zshrc               # User overrides (aliases, editor, pnpm)
├── .p10k.zsh            # Powerlevel10k theme configuration
├── plugin.zsh           # Plugin loader (zinit examples, disabled by default)
├── prompt.zsh           # Custom prompt loader (disabled by default)
├── user.zsh             # Startup commands & OMZ plugin config
├── conf.d/
│   ├── 00-hyde.zsh      # HyDE initialization entry point
│   ├── binds.zsh        # Custom keybindings
│   └── hyde/
│       ├── env.zsh      # Environment variables (XDG, PATH, etc.)
│       ├── prompt.zsh   # Prompt initialization (starship/p10k)
│       └── terminal.zsh # Core HyDE logic, plugins, functions, completions
├── functions/
│   ├── bat.zsh          # Bat (cat replacement) integration
│   ├── bind_M_n_history.zsh  # Alt+1-9 history recall
│   ├── duf.zsh          # Duf (df replacement) integration
│   ├── eza.zsh          # Eza (ls replacement) aliases
│   ├── fzf.zsh          # FZF fuzzy finder functions
│   ├── kb_help.zsh      # ? key help handler
│   └── error-handlers.zsh   # Error handlers & slow load warning
└── completions/
    ├── fzf.zsh          # FZF completion setup
    ├── hyde-shell.zsh   # hyde-shell command completions
    └── hydectl.zsh      # hydectl command completions
```

## 🔍 File-by-File Breakdown

### Hidden Files

#### `.zshenv`
**Purpose:** Entry point for zsh configuration
- Sources all `conf.d/*.zsh` files in order
- Sets up the initial loading chain before `.zshrc` is processed

#### `.zshrc`
**Purpose:** User override file (HyDE-safe)
- HyDE intentionally keeps this minimal to avoid overwriting user configs
- Contains commented example aliases (eza, navigation, package management)
- Sets `EDITOR=code`
- Configures PNPM_HOME path

#### `.p10k.zsh`
**Purpose:** Powerlevel10k theme configuration
- Generated by `p10k configure` wizard
- Defines prompt segments (os_icon, dir, vcs, etc.)
- Configures colors, icons, separators
- Custom git status formatter (`my_git_formatter()`)

#### `.zcompdump`
**Purpose:** Zsh completion dump cache
- Auto-generated by `compinit` for faster completion loading
- Contains 855 command completions

---

### Root Configuration Files

#### `conf.d/00-hyde.zsh`
**Purpose:** HyDE initialization entry point
- Sources `conf.d/hyde/env.zsh` (required)
- Sources `conf.d/hyde/terminal.zsh` (interactive shells only)
- Named `00-` to ensure it loads first

#### `conf.d/binds.zsh`
**Purpose:** Custom keybindings
- Binds Ctrl+Arrow and Alt+Arrow for word navigation
- Binds `forward-word` and `backward-word` to multiple key sequences

#### `plugin.zsh`
**Purpose:** Plugin system configuration (currently disabled)
- Contains `return 1` at top (disabled by default)
- Includes zinit plugin manager setup examples
- Example plugins: history-search-multi-word, zsh-autosuggestions, fzf-tab, etc.
- **To enable:** Comment out `return 1`

#### `prompt.zsh`
**Purpose:** Custom prompt loader (currently disabled)
- Contains `return 1` at top (disabled by default)
- Would load starship or other custom prompts
- **To enable:** Comment out `return 1`

#### `user.zsh`
**Purpose:** User startup commands & OMZ plugin configuration
- Displays pokemon/fastfetch ASCII art on startup
- Configures Oh-My-Zsh plugins: `git`, `zsh-256color`, `zsh-autosuggestions`, `zsh-syntax-highlighting`
- Contains override options (commented):
  - `HYDE_ZSH_NO_PLUGINS=1` - Disable OMZ plugins
  - `HYDE_ZSH_PROMPT` - Disable HyDE prompt
  - `HYDE_ZSH_COMPINIT_CHECK=1` - Set compinit check interval
  - `HYDE_ZSH_OMZ_DEFER=1` - Defer OMZ loading

---

### HyDE Core Files (`conf.d/hyde/`)

#### `conf.d/hyde/env.zsh`
**Purpose:** Environment variable initialization
- Sets up XDG Base Directory spec variables
- Configures PATH with `~/.local/bin`
- Sets application config paths (LESSHISTFILE, PARALLEL_HOME, etc.)
- Exports `HYPRLAND_CONFIG`

#### `conf.d/hyde/prompt.zsh`
**Purpose:** Prompt system initialization
- Auto-detects and loads starship or powerlevel10k
- Controlled by `HYDE_ZSH_PROMPT` variable
- Sets up STARSHIP_CONFIG and STARSHIP_CACHE

#### `conf.d/hyde/terminal.zsh`
**Purpose:** Core HyDE functionality (largest file)
- **Functions:**
  - `_load_functions()` - Sources all `functions/*.zsh`
  - `_load_completions()` - Sources all `completions/*.zsh`
  - `_dedup_zsh_plugins()` - Deduplicates OMZ plugins
  - `_defer_omz_after_prompt_before_input()` - Deferred OMZ loading
  - `_load_deferred_plugin_system_by_hyde()` - Deferred plugin system
  - `do_render()` - Checks terminal image/art support
  - `_load_compinit()` - Optimized completion initialization
  - `_load_prompt()` - Prompt loader
  - `__package_manager()` - HyDE package manager wrapper

- **Environment Variables:**
  - `HYDE_ZSH_DEFER="1"` - Enable deferred loading
  - `HYDE_ZSH_PROMPT="1"` - Enable prompt customization
  - `HYDE_ZSH_NO_PLUGINS="0"` - Plugin loading toggle
  - `PM_COMMAND=(hyde-shell pm)` - Package manager command

- **Aliases:**
  - `c`, `in`, `un`, `up`, `pl`, `pa`, `vc`, `fastfetch`
  - Navigation: `..`, `...`, `.3`, `.4`, `.5`
  - `mkdir='mkdir -p'`

- **History Configuration:**
  - `HISTFILE`, `HISTSIZE=10000`, `SAVEHIST=10000`

---

### Functions

#### `functions/bat.zsh`
**Purpose:** Bat (cat replacement) integration
- Replaces `cat` with `bat --style=plain`
- Adds `--help` piping to bat for formatted help output

#### `functions/bind_M_n_history.zsh`
**Purpose:** Alt+1-9 history recall
- Binds `Alt+1` through `Alt+9` to recall Nth command from history
- Uses `fc -ln` to fetch historical commands

#### `functions/duf.zsh`
**Purpose:** Duf (df replacement) integration
- Wraps `duf` command as `df` alias
- Handles arguments properly

#### `functions/eza.zsh`
**Purpose:** Eza (ls replacement) aliases
- `l` - Long list
- `ll` - Long list all (including hidden)
- `ld` - Long list directories only
- `lt` - Tree view

#### `functions/fzf.zsh`
**Purpose:** FZF fuzzy finder integration
- `_fuzzy_change_directory` - FZF directory navigation
- `_fuzzy_edit_search_file_content` - Search and edit file contents
- `_fuzzy_edit_search_file` - Search and edit files
- `_fuzzy_search_cmd_history` - FZF history search
- Aliases: `ffcd`, `ffec`, `ffe`, `ffch`

#### `functions/kb_help.zsh`
**Purpose:** `?` key help handler
- Appends `--help` to commands and executes
- Smart detection: ignores `?` inside quotes or when buffer is empty

#### `functions/error-handlers.zsh`
**Purpose:** Error handling and diagnostics
- `command_not_found_handler` - Suggests packages for missing commands
- `_slow_load_warning` - Warns if shell takes >3s to load
- `no_such_file_or_directory_handler` - Custom error formatting

---

### Completions

#### `completions/fzf.zsh`
**Purpose:** FZF shell completion
- Evaluates `fzf --zsh` for Ctrl-R completion

#### `completions/hyde-shell.zsh`
**Purpose:** Tab completion for `hyde-shell` command
- Completes built-in commands: `help`, `reload`, `wallbash`, `version`, etc.
- Dynamically fetches hyde scripts via `hyde-shell --list-script`

#### `completions/hydectl.zsh`
**Purpose:** Tab completion for `hydectl` command
- Uses `hydectl completion zsh` for dynamic completions

## 🔧 Key Configuration Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `HYDE_ZSH_DEFER` | `"1"` | Enable deferred Zsh loading |
| `HYDE_ZSH_PROMPT` | `"1"` | Enable HyDE prompt customization |
| `HYDE_ZSH_NO_PLUGINS` | `"0"` | Disable plugin loading when set to `"1"` |
| `HYDE_ZSH_COMPINIT_CHECK` | `1` | Hours between compinit security checks |
| `HYDE_ZSH_OMZ_DEFER` | unset | Defer OMZ loading until after prompt |
| `PM_COMMAND` | `(hyde-shell pm)` | Package manager command |

## 🚀 Loading Order

1. `.zshenv` → Sources `conf.d/*.zsh`
2. `conf.d/00-hyde.zsh` → Sources `hyde/env.zsh` and `hyde/terminal.zsh`
3. `conf.d/hyde/env.zsh` → Sets environment variables
4. `conf.d/hyde/terminal.zsh` → Core logic:
   - Loads compinit
   - Deduplicates plugins
   - Handles deferred OMZ loading
   - Loads functions and completions
   - Sources user config (`.user.zsh` or `user.zsh`)
5. `.zshrc` → User overrides (minimal in HyDE)

---

## 📝 Notes

- HyDE uses **deferred loading** to improve shell startup time
- User configurations should go in `~/.user.zsh` or `$ZDOTDIR/user.zsh`
- To disable HyDE features, unset variables in `user.zsh` before they're used
- The `plugin.zsh` and `prompt.zsh` files are disabled by default (`return 1`)
- HyDE integrates with `hyde-shell` package manager for system packages
